## 출력 형식 지시

위 정보를 바탕으로 다음 형식으로 출력해줘:

1. 각 에이전트마다 별도 섹션으로 구분
2. 각 섹션 제목: ## [파일명].md (예: ## instagram-carousel-generator.md)
3. 파일 전체 내용을 코드블록(```로 감싸서) 안에 출력
4. 코드블록 밖에는 설명 없이 오직 파일 내용만

각 .md 파일은 반드시 아래 구조를 따라야 해:

### 오케스트레이터가 포함된 경우 추가 요구사항:

오케스트레이터 .md 파일에는 반드시:
- PIPELINE OVERVIEW 섹션 (전체 스텝 나열)
- 각 스텝별 EXECUTION INSTRUCTIONS
- HUMAN REVIEW GATE (사용자가 "approve" 입력 전까지 멈추는 단계)
- REVISION LOOP (피드백 시 해당 에이전트부터 재실행)
- HARD RULES 섹션 (최소 5개)
- ERROR HANDLING 섹션

### 시각적 출력 에이전트가 포함된 경우 추가 요구사항:

HTML 생성 에이전트에는 반드시:
- Canvas Specifications 섹션 (width, height, padding — NON-NEGOTIABLE 명시)
- Design System 섹션 (색상 팔레트, 타이포그래피 크기 범위, 레이아웃 원칙)
- html, body { width: [W]px; height: [H]px; overflow: hidden; } 명시
- Google Fonts @import 방식 사용 (외부 CSS 파일 금지)
- File Naming Convention 섹션 (slide_01.html 형식)
- Quality Checklist (자가 검증 항목)

### PNG 내보내기 에이전트가 포함된 경우 추가 요구사항:

내보내기 에이전트에는 반드시:
- Puppeteer 설정 코드블록 포함:
  page.setViewport({ width: [W], height: [H], deviceScaleFactor: 1 })
  waitUntil: 'networkidle0'
  omitBackground: false
- 출력 폴더 네이밍 규칙: /output/[프로젝트명]_[YYYYMMDD]/
- Validation Checklist 표 형식
- 최종 Production Report 형식 (✅ 이모지 포함)
- Puppeteer 없을 때 fallback 3가지 안내

이제 위 모든 조건을 충족하는 완성된 .md 파일들을 출력해줘.

## 작성 예시 (참고용)

아래는 실제로 채워진 예시입니다.

> 기존 인스타 캐러셀(1080x1440 세로 / 마케팅 카피 중심)과 차별화된
> **정사각형 뉴스 에디토리얼 카드뉴스** 스타일입니다.

```
**워크플로우 이름**: 인스타그램 뉴스 카드뉴스 제작 파이프라인

**최종 목표**: 뉴스 주제 입력 → 팩트 리서치 → 카드별 텍스트 작성 → 에디토리얼 HTML 생성 → PNG 내보내기

**에이전트 목록**:
1. news-researcher: 주제를 받아 팩트/통계/인용구/출처를 리서치하고 핵심 데이터 포인트 추출
   → 입력: 주제 + 타겟 독자 / 출력: 검증된 데이터 브리프 (5~8개 핵심 사실)

2. card-scripter: 데이터 브리프를 받아 각 카드의 헤드라인 + 핵심 문장 + 출처 라벨 작성
   → 입력: 데이터 브리프 / 출력: 카드별 텍스트 (헤드라인 / 본문 1~2줄 / 출처)

3. editorial-designer: 카드 텍스트를 받아 매거진 그리드 스타일 HTML 파일 생성
   → 입력: 카드별 텍스트 / 출력: card_01.html ~ card_NN.html

4. card-exporter: HTML 파일을 받아 1080x1080px PNG로 변환 후 폴더 정리
   → 입력: HTML 파일들 / 출력: card_01.png ~ card_NN.png

**오케스트레이터 포함 여부**: 있음
→ news-orchestrator가 위 4개 에이전트를 순서대로 호출하고,
   card-scripter 결과물 출력 후 사용자 승인 게이트를 포함.

---

**캔버스 규격**: 1080x1080px (1:1 정사각형)
**안전 여백**: 80px
**출력 형식**: HTML → PNG

---

**색상 팔레트**: 직접 지정
- 배경: #F5F0E8 (오프화이트 / 신문지 질감 느낌)
- 텍스트: #1A1A1A (딥 블랙)
- 강조: #C8102E (저널리즘 레드 / 속보 느낌)

**타이포그래피**:
- 헤드라인 폰트: Playfair Display (serif — 매거진/신문 느낌)
- 본문 폰트: Inter (sans-serif — 가독성)
- 헤드라인 크기: 72~88px (정사각형이라 세로형보다 작게)
- 본문 크기: 36~44px

**전체 무드**: 에디토리얼하고 신뢰감 있게. 화려한 그래픽보다 타이포그래피와 여백으로 승부.

**레이아웃 특이사항**:
- 각 카드 상단에 카테고리 라벨 (예: [경제] [사회] [테크]) — 36px 볼드 레드
- 카드 번호를 좌측 상단에 크게 표시 (예: 01, 02) — 장식 요소로 활용
- 하단에 출처 라인 고정 (예: 출처: 한국은행, 2025) — 28px 뮤트 컬러
- 통계/숫자는 별도 강조 블록 처리 (배경 레드, 텍스트 화이트)

---

**변환 도구**: Puppeteer
**파일 네이밍**: card_01.png, card_02.png
**출력 폴더**: output/cardnews_주제슬러그_날짜/html/ + png/
```

## 팁

- **에이전트가 3개 이상**이면 오케스트레이터를 꼭 추가하세요. 수동 핸드오프를 없애줍니다.
- **시각적 출력이 없는 워크플로우** (텍스트 분석, 리포트 생성 등)도 동일하게 사용 가능합니다. 디자인/내보내기 섹션만 비워두세요.
- 생성된 파일들은 `~/.claude/agents/` 폴더에 저장하면 바로 사용 가능합니다.
- 오케스트레이터 파일만 먼저 실행하면 나머지 에이전트들이 자동으로 호출됩니다.
